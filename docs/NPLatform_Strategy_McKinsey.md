# I. Situation — 시장은 크고, 디지털화는 비어있다

## 1.1 한국 부실채권·부동산 시장은 ₩215조 규모이며, 그 중 ₩28조가 즉시 디지털 전환 가능하다

**Key Message** — 시장의 절대 규모보다 더 중요한 것은 디지털 전환 가능 영역(SAM)이 명확하다는 점이다.

### Exhibit 1.1 — Market Sizing (2026 기준)

| 구분 | 규모 | 비고 |
|---|---:|---|
| **TAM — 전체 시장** | **₩215조** | NPL ₩15조 + 부동산 직거래 ₩200조 |
| **SAM — 디지털 가능** | **₩28조** | 기관·전문 투자자 거래 |
| **SOM — 5년 차 목표** | **₩2.4조** | 점유율 8.6%, GMV |

### 시장 성장 동인 (Three Tailwinds)

- **만기 도래** — 코로나·금리 인상기에 누적된 부실채권이 2026~2028 집중 만기 도래
- **금리 인하** — 한국은행 기준금리 인하 사이클 진입, 부동산 거래 회복
- **규제 정비** — 금융위 NPL 디지털 매각 가이드라인 시행 (2025 Q4)

> **So What**: 시장은 크고, 자라고, 규제가 정비되고 있다. **타이밍 윈도우는 향후 24개월이다.**

**Key Message** — 매도·매수 양측이 동시에 손해를 보는 구조적 비효율이 5개 영역에서 발생한다.

# II. Complication — 정보 사업 모델로는 락인이 불가능하다

## 2.1 단순 정보 판매는 5개 락인 메커니즘 모두에서 실패한다

**Key Message** — 락인은 의지가 아니라 구조다. 정보 사업은 5개 락인 차원 모두에서 약하다.

### Exhibit 2.1 — Lock-in Diagnostic

| 락인 차원 | 정보 사업 | 거래 사업 (NPLatform) |
|---|---|---|
| **데이터 락인** | 약함 (사용자가 다운받아 떠남) | **강함** (자료는 워터마크·DRM, 다운로드 차단) |
| **프로세스 락인** | 없음 | **강함** (LOI/협상/계약/정산 8단계) |
| **재무 락인** | 약함 (월 구독료 ₩99k) | **강함** (에스크로·크레딧·보증금) |
| **네트워크 락인** | 약함 (1-sided) | **강함** (양면 시장 + Deal Access Score) |
| **규제 락인** | 없음 | **강함** (PII 마스킹·감사 로그·금융위 가이드 준수) |

> **So What**: 5개 차원 모두에서 강한 모델만이 살아남는다. 우리는 **8단계 거래 퍼널** 로 5개 차원을 모두 점유한다.

## 2.2 가장 위험한 안티패턴 3가지가 있다 — 이를 허용하는 순간 비즈니스는 죽는다

**Key Message** — 직관적으로 사용자 친화적으로 보이는 3가지 기능이 사실은 비즈니스를 즉사시키는 함정이다.

### Exhibit 2.2 — Three Forbidden Features

| 절대 금지 ❌ | 직관적 매력 | 진짜 결과 |
|---|---|---|
| **엑셀 다운로드 제공** | "사용자가 편리해함" | 자료가 플랫폼을 벗어나는 순간 거래는 바깥에서 일어난다 |
| **이메일로 자료 전달** | "고객 응대" | 추적 불가 → 직거래 100% 발생 |
| **담당자 직통 연락 허용** | "친절한 서비스" | 매도-매수 직접 연결 → 수수료 회피 |

> **Founder's Rule** — *"이 3개를 허용하는 순간, 우리는 정보 회사가 되고 비즈니스는 죽는다. 어떤 사용자 요청에도 양보하지 않는다."*

# III. Question — 어떻게 락인 가능한 거래 구조를 만들 것인가

## 3.1 핵심 질문은 단 하나다 — **"플랫폼을 떠나는 순간 손해" 를 어떻게 만들 것인가**

### Sub-Questions

1. 매도 기관이 다른 채널 대비 NPLatform에서 더 잘 팔리도록 어떻게 만들 것인가? *(공급 락인)*
2. 매수자가 다른 곳에서 같은 정보를 얻을 수 없도록 어떻게 만들 것인가? *(수요 락인)*

# IV. Answer — 8-Stage Lock-in Architecture

## 4.1 거래의 6 Critical Path를 모두 플랫폼 안에서 발생시킨다

**Key Message** — 정보 → 접근 → 참여 → 거래 → 계약 → 정산. 단 한 단계라도 외부에서 일어나면 전체 락인이 무너진다.

### Exhibit 4.1 — 8-Stage Lock-in Funnel

```
[1] 매각 등록     → AI 가격 가이드·자동 보고서        → "여기서 파는 게 더 잘 팔림"
       ↓
[2] Teaser 공개   → 지역/유형/규모/수익률만 노출       → "관심은 ↑, 거래는 X"
       ↓
[3] 접근 권한     → 인증·NDA·보증금/크레딧             → "아무나 못 들어옴"
       ↓
[4] 딜룸          → 워터마크·열람 로그·다운로드 통제   → ★ 핵심 방어선
       ↓
[5] LOI 제출      → 플랫폼 내 양식만 가능              → 이메일 우회 차단
       ↓
[6] 매칭·협상    → 채팅·연락처 비공개·AI 추천         → 직거래 차단
       ↓
[7] 계약          → 자동 생성·전자서명·보관            → 외부 계약 시 패널티
       ↓
[8] 정산·클로징   → 에스크로·자동 정산                 → 돈 흐름 락인
```

### 단계별 락인 장치 매핑

| Stage | 락인 장치 | 락인 강도 |
|---|---|:---:|
| 1. 매각 등록 | AI 가격 가이드, 자동 보고서, 매도자 백오피스 | ⭐⭐⭐ |
| 2. Teaser | 마스킹·티저 자동 생성 | ⭐⭐ |
| 3. 접근 권한 | NDA·보증금·크레딧·인증 | ⭐⭐⭐⭐ |
| 4. 딜룸 | 워터마크·DRM·열람 로그·다운로드 통제 | ⭐⭐⭐⭐⭐ |
| 5. LOI | 플랫폼 내 폼 제출 강제 | ⭐⭐⭐⭐ |
| 6. 협상 | 채팅 로그·연락처 비공개·AI 매칭 | ⭐⭐⭐⭐⭐ |
| 7. 계약 | 전자서명·자동 생성·보관 | ⭐⭐⭐⭐ |
| 8. 정산 | 에스크로·자동 수수료 | ⭐⭐⭐⭐⭐ |

> **So What**: 락인은 한 곳에서가 아니라 8개 단계 누적으로 만들어진다. 이탈 비용은 각 단계의 곱(multiplicative)으로 증가한다.

## 4.2 4-Tier Access Model로 정보 공개와 락인을 동시에 달성한다

**Key Message** — "담보는 열고, 사람은 가린다." 정보 단계별 노출 범위를 4단계로 통제한다.

### Exhibit 4.2 — 4-Tier Access Model

| Tier | 공개 정보 | 진입 조건 | 락인 효과 |
|:---:|---|---|---|
| **L0**<br>Public | 지역·유형·규모·할인율·AI 등급 | 비회원 가능 | SEO·바이럴·신규 유입 |
| **L1**<br>Verified | + 마스킹된 권리분석 요약 | 본인인증 + 투자자 자격 | 회원 전환·이메일 수집 |
| **L2**<br>NDA | + 감정평가서·등기·세부 권리 | NDA + 자금력 인증 + 보증금/크레딧 | **진성 매수자 필터링** |
| **L3**<br>Deal Room | + 워크아웃 이력·내부 자료·실사 보고서 | LOI 제출 + 매도자 승인 | **완전 락인** (이탈 불가) |

> **Compliance Bonus**: 4단계 모델은 한국 개인정보보호법 + 금융위 NPL 가이드라인을 100% 만족시킨다. **규제는 우리의 해자다.**

| **AI 분석** | 가격 추정·권리 리스크·시세 비교 | AI 크레딧 차감 |
| **실사** | 현장 일정·보고서·사진/동영상 | 하이브리드 워크플로우 (§5.1) |
| **Q&A** | 매수자 ↔ 매도자 질의 | 1:1 채널, 담당자 비공개 |
| **오퍼** | LOI/입찰 제출, 협상 히스토리 | 외부 제출 차단 |
| **계약** | 계약서 자동 생성, 전자서명 | 외부 계약 시 패널티 |
| **정산** | 에스크로 상태·영수증·수수료 | 돈 흐름 100% 플랫폼 内 |

### 4-Layer 보안 스택

```
Layer 1 — Access Control       RLS + JWT + IP allowlist (기관)
Layer 2 — Document Protection  서버사이드 렌더링 + 워터마크 + Signed URL (TTL 5분)
Layer 3 — Behavior Tracking    페이지 view 시간, 마우스, 스크롤, 복사 시도, 스크린샷 감지
Layer 4 — Audit Trail          Append-only log (Postgres + WORM S3)
```

> **So What**: 딜룸은 NPLatform의 심장이다. 이 한 곳이 무너지면 전체가 무너진다. **기술적 자원의 30% 이상을 딜룸에 투입한다.**

## 5.1 모든 것을 온라인으로 끌고 갈 수는 없다 — 하이브리드를 정직하게 인정한다

**Key Message** — 현장 실사와 담당자 미팅은 사라지지 않는다. 우리의 차별화는 이를 부정하는 것이 아니라, 80%를 온라인화하고 20%를 하이브리드로 만드는 것이다.

### 하이브리드 = 온라인 미팅 + 오프라인 미팅의 병행

**핵심 정의**: 하이브리드는 "온라인 자료 + 오프라인 현장"이 아니라, **온라인 미팅과 오프라인 미팅이 동일한 딜룸 안에서 병행 운영되는 것**이다.

```
[1] 사전 자료 검토        딜룸에서 감정평가서·등기·권리·재무 등 자료 열람
        ↓
[2] AI 권리 자동 분석     등기·임차·체납 등 사전 분석으로 의문점 추출
        ↓
[3] 1차 온라인 미팅       딜룸 화상회의 (Zoom/Meet 통합)
                          매도 담당자 ↔ 매수자 Q&A
                          자동 녹화·AI 회의록 → 딜룸 영구 저장
        ↓
[4] 오프라인 현장 미팅    필요 시 플랫폼 통한 일정 예약·동행
                          매수자가 못 갈 경우 검증된 실사 대행 매칭
        ↓
[5] 2차 온라인 후속 미팅  현장 결과 공유, 추가 협상 (다시 화상)
        ↓
[6] 모든 미팅 기록        온라인·오프라인 구분 없이 딜룸 타임라인에 통합
```

### 하이브리드 운영 원칙

| 단계 | 권장 형태 | 이유 |
|---|---|---|
| 1차 인터뷰 | **온라인** | 시간·비용 절감, 진성 매수자 필터링 |
| 자료 명확화 | **온라인** | 화면 공유로 즉시 해결 |
| 현장 확인 | **오프라인** | 물리적 검증 필요 |
| 담당자 미팅 | **온라인 또는 오프라인** | 매도 기관 선호에 맞춤 |
| 후속 협상 | **온라인** | 빠른 의사결정 |
| 계약 서명 | **온라인 (전자서명)** | 100% 비대면 가능 |

> **So What**: 모든 일정·연락·기록은 플랫폼을 통해서만 일어난다. 오프라인 미팅을 인정하되, **그 주변(예약·녹화·후속)은 100% 온라인에 묶어둔다.**

## 7.2 가장 강력한 락인은 Deal Access Score 라는 사용자 자산화다

**Key Message** — 점수가 자산이 되면 사용자는 떠날 수 없다. 게임의 SBT(Soulbound Token)와 같은 원리다.

### Deal Access Score 구성 요소

```
점수 (0~1000) =
  거래 완료 건수      × 20
+ 평균 입찰 신뢰도    × 10
+ LOI 진성 비율       × 30
+ 결제·정산 신용      × 20
+ 커뮤니티 기여       × 5
```

### 점수의 자산화 효과

- 高점수 → **좋은 딜 24h 우선 공개** (Exclusive Window)
- 高점수 → **수수료 할인** (1단계당 -0.1%)
- 高점수 → **보증금 면제**
- 점수는 **이전 불가능** → 떠나면 0부터 다시

> **So What**: Deal Access Score는 전환 비용을 무한대로 만든다. **이것이 NPLatform이 단명 정보 사이트와 다른 결정적 이유다.**

## 10.3 Founder's Principles — 8 Commandments

> 1. **거래가 일어나지 않는 기능은 만들지 않는다.**
> 2. **정보를 다운로드 가능하게 하면 회사가 죽는다.**
> 3. **AI는 도구일 뿐, 신뢰의 원천은 데이터와 프로세스다.**
> 4. **디자인은 신뢰의 90%다. 톱티어가 아니면 출시하지 않는다.**
> 5. **하이브리드를 인정하라 — 현장은 사라지지 않는다. 단지 최소화될 뿐.**
> 6. **양면 시장은 한쪽이 무너지면 끝난다 — 매도 기관에 모든 것을 걸어라.**
> 7. **규제는 해자다. 두려워하지 말고 끌어안아라.**
> 8. **단순 정보 사업은 절대 하지 않는다.**
