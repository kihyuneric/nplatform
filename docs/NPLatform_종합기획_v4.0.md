# NPLatform 종합기획 v4.0 — 400점 마스터플랜

## 1. 비전 & 비즈니스 모델

### 1.1 플랫폼 비전
> "국내 최초 AI Agent 기반 NPL 종합 투자 플랫폼"
> 금융기관, 파트너사, 투자자를 연결하는 NPL 거래 생태계

### 1.2 수익 모델 (Revenue Share)

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│  금융기관    │────▶│  NPLatform   │◀────│  투자자     │
│  (매도자)    │     │  (플랫폼)    │     │  (매수자)   │
└─────────────┘     └──────┬───────┘     └─────────────┘
                           │
              ┌────────────┼────────────┐
              ▼            ▼            ▼
        ┌──────────┐ ┌──────────┐ ┌──────────┐
        │자산운용사│ │대부업체  │ │공인중개사│
        │(AMC)     │ │          │ │          │
        └──────────┘ └──────────┘ └──────────┘
              파트너사 (영업·마케팅·컨설팅)
```

### 1.3 수익쉐어 구조

| 수익원 | 비율 | 플랫폼 | 파트너사 | 비고 |
|--------|------|--------|----------|------|
| **입찰 수수료** | 거래금액 0.3~1% | 60% | 40% | 파트너 소개 건 |
| **프리미엄 구독** | 월 99,000~999,000원 | 70% | 30% | 파트너 유치 건 |
| **AI 분석 리포트** | 건당 50,000~200,000원 | 65% | 35% | 파트너 추천 건 |
| **광고·배너** | 노출/클릭 기반 | 80% | 20% | 파트너 영역 |
| **컨설팅 수수료** | 프로젝트 기반 | 50% | 50% | 공동 진행 건 |

### 1.4 파트너 유형별 역할

**자산운용사 (AMC)**
- NPL 포트폴리오 소싱·실사 지원
- 금융기관 대상 입찰 대행
- 수익: 거래 수수료 쉐어 + 실사비

**대부업체**
- NPL 담보물건 자금 조달
- 소액 NPL 직접 매입
- 수익: 매입건 수수료 + 금리 수익

**공인중개사**
- 담보물건 현장 실사
- 매각 후 임차인 관리
- 수익: 중개 수수료 + 실사비

## 6. DB 스키마 확장

### 새 테이블

```sql
-- 파트너사
CREATE TABLE partners (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(user_id),
  partner_type TEXT NOT NULL CHECK (partner_type IN ('AMC', 'LENDER', 'AGENT')),
  company_name TEXT NOT NULL,
  business_number TEXT,
  representative TEXT,
  phone TEXT,
  address TEXT,
  specialties TEXT[], -- 전문분야
  regions TEXT[], -- 활동지역
  status TEXT DEFAULT 'PENDING' CHECK (status IN ('PENDING','APPROVED','REJECTED','SUSPENDED')),
  commission_rate DECIMAL(5,2) DEFAULT 40.00, -- 수수료율 %
  created_at TIMESTAMPTZ DEFAULT now()
);

-- 리드
CREATE TABLE partner_leads (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  partner_id UUID REFERENCES partners(id),
  listing_id UUID REFERENCES npl_listings(id),
  lead_type TEXT CHECK (lead_type IN ('AUTO','MANUAL','REFERRAL')),
  status TEXT DEFAULT 'NEW' CHECK (status IN ('NEW','ACCEPTED','IN_PROGRESS','COMPLETED','REJECTED')),
  assigned_at TIMESTAMPTZ DEFAULT now(),
  completed_at TIMESTAMPTZ,
  notes TEXT
);

-- 정산
CREATE TABLE settlements (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  partner_id UUID REFERENCES partners(id),
  period_start DATE NOT NULL,
  period_end DATE NOT NULL,
  total_revenue BIGINT DEFAULT 0,
  platform_share BIGINT DEFAULT 0,
  partner_share BIGINT DEFAULT 0,
  status TEXT DEFAULT 'PENDING' CHECK (status IN ('PENDING','APPROVED','PAID')),
  invoice_url TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- AI 대화
CREATE TABLE ai_conversations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(user_id),
  title TEXT,
  messages JSONB DEFAULT '[]',
  context JSONB DEFAULT '{}',
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

## 7. AI Agent 설계

### 7.1 Agent 아키텍처

```
사용자 프롬프트
      ↓
┌─────────────────┐
│  Intent Classifier│ → 검색 / 분석 / 상담 / 일반
└────────┬────────┘
         ↓
┌─────────────────┐
│  Tool Selection  │ → search_npl / analyze_npl / simulate / chat
└────────┬────────┘
         ↓
┌─────────────────┐
│  Tool Execution  │ → Supabase Query / AI Analysis / Calculator
└────────┬────────┘
         ↓
┌─────────────────┐
│  Response Gen    │ → 자연어 + 카드/차트/테이블 렌더링
└─────────────────┘
```

### 7.2 지원 프롬프트 예시

| 카테고리 | 프롬프트 | 응답 형태 |
|----------|----------|-----------|
| 검색 | "서울 강남 아파트 NPL 검색" | 매물 카드 리스트 |
| 분석 | "이 매물 AI 분석해줘" | 분석 리포트 카드 |
| 시뮬 | "감정가 5억, 낙찰가율 70% 수익률" | 시뮬레이션 결과 |
| 통계 | "이번달 NPL 시장 동향" | 차트 + 요약 |
| 상담 | "NPL 투자 처음인데 어떻게 시작?" | 가이드 텍스트 |

### 11.3 문서 OCR 파이프라인

```
문서 업로드 (PDF/이미지)
    ↓
┌─────────────────┐
│ AI 문서 분류     │ → 감정평가서? 등기부? 경매문서? 기타?
└────────┬────────┘
         ↓

│ OCR 텍스트 추출  │ → Tesseract / Claude Vision
└────────┬────────┘
         ↓
┌─────────────────┐

└────────┬────────┘
         ↓
┌─────────────────┐

         ↓
┌─────────────────┐
│ 매물 자동 등록   │ → npl_listings INSERT
└─────────────────┘
```

### 11.4 자동 분석 파이프라인

매물 등록 즉시 트리거:
1. **AI 등급 산정** → A/B+/B/C/D 자동 부여
2. **리스크 점수** → 0-100 다차원 계산
3. **유사사례 검색** → 동일 지역/유형 낙찰사례 5건 매칭
4. **시장가 비교** → KB시세, 실거래가, 공시가 비교
5. **수익성 시뮬** → 기본 시나리오 자동 실행

### 11.5 자동 마케팅 배포

매물 등록 + AI분석 완료 후:
1. **파트너 자동알림** → 지역/전문분야 매칭 파트너에게 푸시
2. **투자자 매칭** → 수요설문 매칭 투자자에게 알림
3. **뉴스레터 포함** → 주간 뉴스레터 신규매물 섹션
4. **지도 자동반영** → 카카오맵 마커 실시간 추가
5. **통계 업데이트** → 시장 통계 자동 갱신

### 11.6 포트폴리오 대시보드 (기관 전용)

```
/institution/portfolio
├── 업로드 이력 (날짜, 파일명, 건수, 상태)
├── 매물 현황 (전체/진행중/입찰중/완료)
├── 일괄 수정 (체크박스 선택 → 일괄 가격변경)
├── 일괄 삭제 (체크박스 선택 → 확인 후 삭제)
├── 통계 요약 (총 매물, 총 채권액, 평균 할인율)
└── AI 분석 현황 (등급 분포, 평균 리스크)
```

## 12. 확장 DB 스키마

```sql
-- 대량 업로드 이력
CREATE TABLE bulk_uploads (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(user_id),
  institution_id UUID,
  file_name TEXT NOT NULL,
  file_size BIGINT,
  file_type TEXT CHECK (file_type IN ('EXCEL', 'PDF', 'IMAGE')),
  total_rows INT DEFAULT 0,
  success_rows INT DEFAULT 0,
  error_rows INT DEFAULT 0,
  status TEXT DEFAULT 'PROCESSING' CHECK (status IN ('PROCESSING','COMPLETED','FAILED','PARTIAL')),
  error_log JSONB DEFAULT '[]',
  column_mapping JSONB, -- 사용자 지정 컬럼 매핑
  created_at TIMESTAMPTZ DEFAULT now(),
  completed_at TIMESTAMPTZ
);

-- 자동 분석 큐
CREATE TABLE analysis_queue (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  listing_id UUID REFERENCES npl_listings(id),
  priority INT DEFAULT 5,
  status TEXT DEFAULT 'PENDING' CHECK (status IN ('PENDING','PROCESSING','COMPLETED','FAILED')),
  result JSONB,
  error_message TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  processed_at TIMESTAMPTZ
);

-- 마케팅 알림 로그
CREATE TABLE marketing_notifications (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  listing_id UUID REFERENCES npl_listings(id),
  recipient_type TEXT CHECK (recipient_type IN ('PARTNER','INVESTOR','NEWSLETTER')),
  recipient_id UUID,
  channel TEXT CHECK (channel IN ('EMAIL','PUSH','SMS','IN_APP')),
  status TEXT DEFAULT 'PENDING',
  sent_at TIMESTAMPTZ,
  opened_at TIMESTAMPTZ
);
```
