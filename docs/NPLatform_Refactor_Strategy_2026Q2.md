## 0. Executive Summary

현재 NPLatform 은 F1–F7 공용 폼 모듈화로 매물등록·입찰등록·NPL분석 3모드를 단일 `UnifiedFormState` 에 통합했다. 그러나 다음 5개 구조적 부채가 Deal Flow 전환율·신뢰도·사용자 인지 부하를 저해한다.

| # | 부채 | 영향 받는 페르소나 | 현 상태 | 목표 상태 |
|---|------|---------------------|---------|-----------|
| D1 | 특수조건 25항목이 입력·분석·리포트 3곳에 그대로 노출 | 매각사·매수자·투자자 | 과잉 정보·의미 겹침 | 입력은 18개 선택, 리포트는 결과만 |
| D2 | 5-팩터 리스크 모델에 "법적"·"권리관계" 중복 | 투자자 | 중복 팩터로 가중치 혼선 | 4-팩터 (법적 → 권리관계에 흡수) |
| D3 | AI 리스크 등급이 특수조건을 종합점수에만 간접 반영 | 투자자 | "AI 프롬프트 이상" | 특수조건 점수 직접 반영 + 설명가능 |
| D4 | 질권대출 LTV 75% 하드코딩 (개인/법인 무차별) | 매입사·투자자 | 회계 관점 오차 | 개인 75% / 법인 90% 분기 기본값 |

## 1. 관점별 현상 진단

### 1.1 🧭 전략기획 관점 — Deal Flow 전환율 저해

**관찰 지표 (추정)**
- 매물등록 Step3(채권정보) → Step4(특수조건) 이탈률 ↑ : 25항목 체크리스트 길이로 인한 피로
- 분석 보고서 "법적 팩터 95점" 소구도 ↓ : "25개 특수조건 전부 해당 없음" 자체가 신뢰 시그널로 작동 안 함
- 매각사 재방문율 ↓ : 등록 후 수정 경로 부재 → 컨택 채널로 문의 우회

**전략 명제**
> **Deal Flow 의 병목은 '정보량'이 아니라 '정보의 의미적 구조'다.**
> 25항목을 3-카테고리(소유권 · 비용 · 유동성)로 재분류하면 매각사는 3초 안에 자기 매물의 리스크 성격을 인지하고, 투자자는 "어디서 돈이 샐 수 있는가"를 버킷 단위로 질문할 수 있다.

**KPI 설계 원칙**
| 단계 | 기존 KPI | 개정 KPI |
|------|---------|---------|
| 매물등록 완료율 | 전체 단계 통과율 | **Step 단위 중도이탈률** + 특수조건 체크 평균 개수 |
| 분석 보고서 신뢰도 | 단순 열람 수 | **"계산식 펼치기" 클릭률** + 팩터별 체류 시간 |
| 매각사 잔존율 | D30 리텐션 | **등록 후 수정 이벤트 발생률** (수정 UX 가용성 지표) |

### 1.2 💼 NPL 도메인 관점 — 25항목 스키마의 세 가지 문제

**문제 1: 의미 중복**
- `선순위 근저당 / 선순위 지상권 / 선순위 임차권 / 선순위 전세권 / 선순위 가등기 / 선순위 가처분 / 선순위 가압류` 7개는 모두 **소유권 우선순위 위협**이라는 단일 리스크 성격을 갖는다. 투자자 관점에서는 "**선순위 등기권리 존재(Y/N)**" 한 항목으로 충분하며, 어떤 선순위인지는 등기부 원본에서 확인하는 것이 정석이다.
- `조세 / 당해세 / 4대보험 미납` 3개도 **법정 우선 공과금** 이라는 단일 카테고리. 당해세는 별도 분리(감액 폭이 특히 크므로).

**문제 2: 점수 입력의 인지 부담**
- 현 체크리스트는 선택과 동시에 감점이 숫자로 표시된다. 매각사는 감점이 무서워 체크를 기피하거나, 반대로 분석자가 재검토해야 할 항목을 선언적으로 놓친다.
- **해결**: 입력 단계에서는 점수 숨김, 리포트 단계에서만 공개.

**문제 3: "법적 팩터" 의 개념 정의 오염**
- 현재 `computeLegalFactor()` 는 `95 − Σ(특수조건 legalPenalty)` 만 계산한다. 반면 `computeRightsFactor()` 는 특수조건 중 임차·선순위 항목을 **다시** 반영한다. 동일 항목이 두 팩터에서 이중 계산되는 구조.
- **NPL 실무 정의**: "법적 리스크"는 "권리분석 리스크"와 동의어다. 별도 팩터로 분리할 이유가 없다. 법적 팩터를 삭제하고 **권리관계 팩터가 18-항목을 단일 소비** 하도록 병합한다.

**재분류 결과 (25 → 18 × 3-버킷)**

| 버킷 | 항목 수 | 성격 | 대표 항목 |
|------|--------|------|----------|
| 🔴 소유권 리스크 | 5 | 소유권 자체가 이전 안 되거나 경합하는 리스크 | 선순위 등기권리·대항력 있는 임차인·유치권 |
| 🟠 비용 리스크 | 7 | 낙찰 후 별도 비용·채무 인수가 발생 | 당해세·임금채권·임차권 등기·조세/4대보험 |
| 🟡 유동성 리스크 | 6 | 매각·임대가 막히거나 지연되는 리스크 | 무허가건축물·맹지·분묘기지권 |

### 1.3 🧮 회계 관점 — 질권대출 LTV 개인 75% / 법인 90% 분기의 논리

**왜 분기가 필요한가**
질권대출(NPL 매입 금융)은 담보가 "채권 그 자체" 이며, 대주는 채권 회수 확실성을 LTV 로 흡수한다. 회수 확실성은 채무자 유형에 따라 실무상 유의미하게 갈린다.

| 채무자 유형 | 회수 안정성 요인 | 실무 질권 LTV |
|------------|------------------|---------------|
| **개인** | 개인파산·면책·회생으로 회수 기간 불확실. 직장·소득 변동 리스크. 집행 절차에서 경매 유찰 반복 빈도 ↑ | **70–75%** (업계 표준) |
| **법인** | 법인 재무제표·담보재산 평가 용이. 법정관리·워크아웃 등 구조화 절차로 회수 일정 예측 가능. 집행 대상이 다분화 (예금·매출채권·부동산 병행 가능) | **85–90%** (업계 표준) |

**재무 모델링 반영**
분석 보고서의 기대 수익률(IRR)·매입가 상한 계산은 "매입가 ≤ 채권잔액 × 질권 LTV + 자기자본" 제약식을 쓴다. 개인·법인을 동일 75%로 두면:
- **법인 채권**: 투자자가 실제 조달 가능한 자금 대비 **보수적 추정** → 매입가 제안이 낮아져 입찰 경쟁에서 탈락
- **개인 채권**: 실제 대주 승인 LTV 보다 **공격적 추정** → 자금 미확정 상태로 낙찰 → 잔금 조달 실패

즉 **디폴트 75% 는 양쪽 모두 오답**이다.

**적용 범위**
- ✅ 분석 보고서 / NPL 분석 입력폼 (투자자·매입사 관점)
- ❌ 매물 등록 / 입찰 등록 (매각사·채권자 관점 — 관심 밖)

### 1.4 🎨 디자인 관점 — 정보 과잉에서 탐색 가능성으로

**현 UX 문제**
- 분석 보고서에서 5개 팩터 카드가 세로로 나열 → 스크롤 피로
- 법적 팩터 카드 하단에 "25개 특수조건 전부 해당 없음" 과 같은 원시 텍스트 노출
- 체크리스트가 카테고리 헤더 없이 한 줄에 쭉 나열된 경우 (`SpecialConditionsSection`) 선택의 단서가 없음

**디자인 원칙**
1. **Disclosure by Default**: 모든 계산식은 접혀 있고, 사용자가 "계산식" 클릭 시 펼침. 기본은 결과 숫자 + 1줄 해석만.
2. **Categorical Navigation Over Linear Scroll**: 3-버킷을 탭으로 구성하면 (a) 폼 길이 감소, (b) 리포트에서 리스크 성격별 탐색 가능.
3. **Consistent Color Semantics**: 🔴 소유권(치명) · 🟠 비용(중) · 🟡 유동성(경미) 색상을 입력폼·리포트·리스트뷰에 일관 적용.
4. **Edit Affordance Visible**: 매각사 마이페이지에서 등록한 매물 카드마다 "수정" 아이콘을 기본 노출 (감춤 메뉴 X).

**3-탭 UX 설계**

```
┌──────────────────────────────────────────────┐
│ [🔴 소유권] [🟠 비용] [🟡 유동성]              │  ← 탭 헤더
├──────────────────────────────────────────────┤
│ 선택된 탭의 항목만 표시                       │
│ ☐ 전세권만 매각                               │
│ ☐ 선순위 등기권리 존재                        │
│ ☐ 대항력 있는 임차인                          │
│ ☐ 유치권 / 법정지상권                         │
│ ☐ 지분입찰                                    │
└──────────────────────────────────────────────┘
```

- 입력폼: 점수 **숨김**, 체크만
- 리포트: 점수 **공개**, 체크된 항목 + 버킷별 감점 합계

### 2.2 4-팩터 리스크 모델 (5 → 4)

| 팩터 | 역할 | 가중치 | 주 데이터 소스 |
|------|------|-------|----------------|
| 담보가치 | LTV 기반 원가 매력도 | **35%** | 채권잔액 / min(감정가, AI시세) |
| 권리관계 | 18항목 + 등기부 | **30%** | 법적점수 (신규 공식) + 등기부 신호 |
| 시장 | 지역·낙찰가율 | **25%** | 블렌드 낙찰가율 + 지역트렌드 |
| 유동성 | 매각 소요 기간 | **10%** | 과거 매각일수 + 입찰자 수 |

**핵심 변경**
- "법적" 팩터 제거, "권리관계" 팩터가 18항목을 **유일하게** 소비
- 기존 `computeLegalFactor()` 는 `computeRightsFactor()` 에 병합
- 가중치 합 = 100% (기존 `0.30+0.15+0.20+0.10+0.25 = 1.00` 에서 재배분)

**종합 점수 → 등급 테이블**

```
≥ 85  : A  (LOW)       — 우량, 즉시 입찰 가능
≥ 70  : B  (LOW)       — 양호, 표준 실사 후 입찰
≥ 55  : C  (MEDIUM)    — 조건부, 추가 검증 필수
≥ 40  : D  (HIGH)      — 고위험, 대폭 할인 필요
<  40 : E  (CRITICAL)  — 비추천
```

### 2.3 개인/법인 질권대출 기본값 분기

**디폴트 룰 (매입가 상한 = 채권잔액 × 질권LTV + 자기자본)**

```ts
export const DEFAULT_MARGIN_LTV = {
  INDIVIDUAL: 0.75,  // 개인 채무자
  CORPORATE:  0.90,  // 법인 채무자
} as const

export function getDefaultMarginLtv(
  debtorType: "INDIVIDUAL" | "CORPORATE" | null
): number {
  if (debtorType === "CORPORATE") return 0.90
  if (debtorType === "INDIVIDUAL") return 0.75
  return 0.75  // 미입력 시 보수적
}
```

**적용 경로**
| 페이지 | 적용 | 이유 |
|--------|------|------|

## 3. 계산식 & 스코어링 SSoT (투명성)

### 3.1 권리관계 팩터 (법적 팩터 흡수 후)

```
권리관계점수 = max(20, 100 − Σ(checked.penalty))

여기서 checked.penalty 는 SPECIAL_CONDITIONS_V2 18항목 중 매물이 선택한 것만.
```

### 3.2 AI 리스크 등급 재설계

**기존 문제 (샘플에서 지적된 부분)**

```
리스크점수 = LTV×0.35 + 지역×0.25 + 낙찰가율×0.30 + 종합회수×0.10
         = 65×0.35 + 54.4×0.25 + 70×0.30 + 63.3×0.10
         = 77.2점
```

- LTV 65 (?) · 지역 54.4 · 낙찰가율 70 · 종합회수 63.3 → 77.2 : 가중평균 수식 자체는 맞지만 **특수조건(법적)이 입력 변수에 없다**
- 사용자가 "특수조건 25개 반영" 이라고 표시되는데 정작 공식에는 반영 안 됨 → 프롬프트 신뢰 상실

**재설계 공식 (4-팩터 통합)**

```
종합점수 = 담보가치 × 0.35
       + 권리관계 × 0.30      ← 18항목 직접 반영 (100 − Σpenalty)
       + 시장   × 0.25
       + 유동성 × 0.10

예시 (샘플 매물 기준)
  담보가치 = 64.5    (LTV 85.5% → 100 − max(0, 85.5 − 50) = 64.5)
  권리관계 = 85      (당해세 -20 + 일반채권 가압류 흡수 등 -15 = -35 → 100 - 35 = 65; 실제 체크 항목이 2개면 85 근처)
  시장    = 73.1    (블렌드 낙찰가율 83.5%, 지역트렌드 60)
  유동성  = 79      (매각 285일 · 입찰자 1명 가정)

  종합 = 64.5×0.35 + 85×0.30 + 73.1×0.25 + 79×0.10
      = 22.58 + 25.50 + 18.28 + 7.90
      = 74.26 → 등급 B (LOW)
```

**AI 총평 프롬프트 재정의**
- 이전 문구 "25개 특수조건 전부 해당 없음 — 등기/물권/조세 청결" 제거
- 대체: "체크된 리스크 {N}건, 그 중 소유권 {Na}·비용 {Nb}·유동성 {Nc}. {가장 큰 감점 항목 이름}이 {Xpt} 감점으로 지배적."
