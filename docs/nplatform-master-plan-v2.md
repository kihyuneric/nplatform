> 작성일: 2026-03-19
> 기준: marketplace.zip 원본 코드 + 0-news 현재 코드 통합 분석

---

## 1. 비전 & 목표

### 1.1 서비스 정의
**NPLatform(엔플랫폼)**은 금융기관의 NPL(부실채권) 매각과 투자자의 매입을 연결하는
**국내 유일 NPL 종합 거래 플랫폼**이다.

### 1.2 핵심 가치 제안
| 대상 | 가치 |
|------|------|
| **금융기관(매각사)** | 입찰 공고 → 투자자 모집 → 낙찰 → 계약까지 원스톱 |
| **기관 투자자(매입사)** | 통합 검색 → AI 분석 → 수익률 시뮬레이션 → 입찰 참여 |
| **개인 투자자** | NPL 학습 → 매물 탐색 → 관심등록 → 전문가 연결 |

### 1.3 경쟁 우위
1. **OCR 기반 자동 데이터 수집** — 등기부등본, 감정평가서 자동 파싱
2. **AI 분석 엔진** — 낙찰가율, 수익률, 리스크 점수 자동 산출
3. **금융기관 직접 입찰** — 중개인 없이 기관 ↔ 투자자 직거래
4. **딜룸** — NDA 서명 → 실사 → 입찰 → 계약까지 원스톱 진행

### 3.1 NPL 검색 (/market/search) — ★★★ 최우선

#### 3.1.1 기존 marketplace 기능 (반영 대상)
| 기능 | marketplace 원본 | 현재 0-news | 고도화 방향 |

| AI 투자 등급 | A/B/C/D/F 등급 배지 |

### 3.3 NPL 입찰 (/market/bidding) — ★★★★ 최핵심

> **금융기관이 쉽게 올리고, 투자자가 쉽게 참여하는 입찰 장터**

#### 3.3.1 금융기관(매각사) 입찰 등록 플로우
```
1. 매물 등록 (/institution/listings/new)
   ├─ 기본정보: 채권명, 담보주소, 담보유형
   ├─ 금액정보: 대출잔액, 감정가, 설정금액, 희망매각가
   ├─ 입찰조건: 입찰시작일, 마감일, 최저입찰가, 보증금율
   ├─ 첨부서류: 채권명세서, 감정평가서, 등기부등본
   └─ 공개설정: TEASER(기본) / NDA 필요 / 전체 공개

2. 입찰 관리 (/institution/bidding)
   ├─ 진행중 입찰 목록
   ├─ 참여자 현황 (NDA서명/입찰참여/제안금액)
   ├─ 입찰 마감 → 낙찰자 선정
   └─ 계약 진행 → 딜룸 생성
```

#### 3.3.2 투자자(매입사) 입찰 참여 플로우
```
1. 매물 발견 (검색/지도/입찰목록)
2. NDA 서명 (전자서명, IP 기록)
3. 상세정보 열람 (마스킹 해제)
4. 실사 요청 (실사자료 다운로드)
5. Q&A (매각사에 질문)
6. 입찰 참여
   ├─ 입찰금액 입력 (최저가 이상)
   ├─ 보증금: 입찰금액 × 10%
   ├─ 보증보험증권 업로드
   └─ 제출 → 확인 모달 → 완료
7. 결과 확인 → 낙찰 시 딜룸 이동
```

### 3.4 NPL 분석 (/npl-analysis) — ★★★

#### 3.4.1 marketplace 분석 기능 통합
| 분석 항목 | marketplace 원본 | 구현 방식 |
|-----------|------------------|-----------|
| 낙찰가율 통계 (1년/6개월/3개월) | ✅ | → 차트 + 테이블 |
| 법원 정보 & 기일 간격 | ✅ | → 타임라인 UI |
| 유사 낙찰 사례 (반경 검색) | ✅ | → 지도 + 테이블 |
| 실거래가 사례 (1km) | ✅ | → 지도 + 테이블 |
| 엑셀 내보내기 | ✅ | → XLSX 다중 시트 |
| AI 투자 등급 | 🆕 신규 | → A~F 등급 자동 산출 |
| 리스크 체크리스트 | 🆕 신규 | → 자동 진단 항목 |

#### 3.4.2 분석 페이지 구조
```
1. 분석 대상 선택 (사건번호 입력 or 검색에서 이동)
2. 분석 결과 대시보드
   ├─ [요약] AI 등급, 추천 입찰가, 예상 수익률
   ├─ [통계] 낙찰가율 차트, 기간별 비교
   ├─ [낙찰사례] 유사물건 테이블 + 지도
   ├─ [거래사례] 실거래가 테이블 + 지도
   └─ [리스크] 자동 체크리스트 + 점수
3. 내보내기 (PDF/XLSX)
```

### 4.2 RLS 정책
```sql
-- 관심매물: 본인 것만 CRUD
CREATE POLICY favorites_own ON npl_favorites
  USING (user_id = auth.uid());

-- 조회로그: 본인 것만 읽기, 누구나 쓰기
CREATE POLICY view_logs_insert ON npl_view_logs
  FOR INSERT WITH CHECK (true);

-- AI분석: 본인 것만 읽기
CREATE POLICY ai_analyses_own ON npl_ai_analyses
  USING (user_id = auth.uid());

-- 입찰: 매각사는 자기 매물 입찰 전체 조회, 매입사는 본인 입찰만
CREATE POLICY bids_seller ON npl_bids
  FOR SELECT USING (
    EXISTS (SELECT 1 FROM npl_listings
            WHERE id = npl_bids.listing_id
            AND seller_id = auth.uid())
  );
```
