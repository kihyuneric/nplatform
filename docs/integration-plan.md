# NPLatform 서비스 통합 계획

## 핵심 문제: 기능이 흩어져 있고 연결이 안 됨

### 현재 상태 (분리된 섬들)
```
[NPL 마켓] ──X── [딜 브릿지] ──X── [전문가]
     │                │                │
     X                X                X
     │                │                │
[교육과정]      [뉴스(외부)]      [커뮤니티]
```

### 목표 상태 (통합 생태계)
```
                    [NPL 마켓]
                   /    |    \
         [검색/지도]  [입찰]  [펀드/대출]
                   \    |    /
                  [딜 브릿지]  ←→  [전문가 연결]
                   /    |    \
            [실사] [계약/OCR] [AI분석]
                        |
              [교육/뉴스/커뮤니티] (내부 통합)
```

---

## 통합 1: NPL 마켓 ↔ 딜 브릿지 완전 연결

### 현재 문제
- `/market/search`에서 매물 클릭 → 별도 상세 페이지 (딜 브릿지와 무관)
- `/exchange`에서 매물 클릭 → 딜 브릿지 상세 (마켓과 무관)
- 사용자가 두 곳에서 같은 매물을 다르게 봄

### 해결
- **마켓 매물 상세 → 딜 브릿지로 이동 버튼** 추가
  - `/listings/[id]` 페이지에 "딜 브릿지에서 거래하기" CTA
  - `/market/search` 결과에서 "관심 표명" → 딜 브릿지 거래 시작
- **딜 브릿지 메인에 마켓 검색 통합**
  - `/exchange` 페이지에 마켓 필터 통합 (지역/담보유형/금액)
- **통합 매물 ID**
  - 마켓 매물 = 딜 브릿지 매각공고 (같은 데이터)

### 수정 파일
- `app/(main)/listings/[id]/page.tsx` — "딜 브릿지에서 거래" 버튼
- `app/(main)/market/search/page.tsx` — "관심 표명" 버튼 → /exchange/deals
- `app/(main)/exchange/page.tsx` — 마켓 필터 통합

## 통합 2: 교육과정 세부 구현 + 관리자 CRUD

### 현재 문제
- `/knowledge/courses` — 목록만 있고 세부 콘텐츠 없음
- 교육과정 등록/수정 관리자 기능 없음

### 해결
**사용자 페이지:**
- `knowledge/courses/page.tsx` — 과정 목록 (카테고리별 필터)
- `knowledge/courses/[id]/page.tsx` — **신규** 과정 상세 (커리큘럼, 진행률, 수강)
- `knowledge/courses/[id]/lesson/[lessonId]/page.tsx` — **신규** 강의 상세 (텍스트/비디오/퀴즈)

**관리자 페이지:**
- `admin/courses/page.tsx` — **신규** 교육과정 CRUD
- `admin/courses/[id]/page.tsx` — **신규** 과정 편집 (레슨 추가/수정/삭제)

**API:**
- `api/v1/courses/route.ts` — 과정 CRUD
- `api/v1/courses/[id]/lessons/route.ts` — 레슨 CRUD

## 통합 3: 용어사전 세부 구현 + 관리자 CRUD

### 현재 문제
- `/knowledge/glossary` — 기본 목록만, 관리 불가

### 해결
- `knowledge/glossary/page.tsx` — 강화: 검색, 카테고리, 초성 필터
- `knowledge/glossary/[term]/page.tsx` — **신규** 용어 상세 (설명, 관련 용어, 예시)
- `admin/glossary/page.tsx` — **신규** 용어 CRUD
- `api/v1/glossary/route.ts` — 용어 CRUD

## 통합 4: 뉴스 내부 통합

### 현재 문제
- 뉴스 클릭 → 외부 사이트로 이탈
- NPLatform 안에서 기사를 읽을 수 없음

### 해결
- `app/news/page.tsx` — 뉴스 목록 (내부에서 표시)
- `app/news/[id]/page.tsx` — **신규** 뉴스 상세 (본문 내부 표시, 원문 링크)
- 외부 API에서 뉴스 수집 → DB 저장 → 내부 표시
- 관련 매물 자동 연결 (기사 키워드 ↔ 매물 지역/유형)
- `admin/news/page.tsx` — **신규** 뉴스 관리 (외부 수집 + 직접 작성)
- `api/v1/news/route.ts` — 뉴스 CRUD + 외부 수집

## 통합 5: 전문가 ↔ 거래 연결

### 현재 문제
- 전문가 서비스가 거래와 분리됨
- 실사 중 전문가 상담이 필요한데 연결 안 됨

### 해결
- 실사 체크리스트에 "전문가 상담 요청" 버튼
- 딜룸에서 "전문가 초대" 기능
- 매물 상세에 "관련 전문가" 추천 섹션

### 수정 파일
- `exchange/due-diligence/[id]/page.tsx` — 전문가 연결 버튼
- `exchange/deals/[id]/page.tsx` — 전문가 초대
- `exchange/[id]/page.tsx` — 관련 전문가 섹션

---

## 통합 6: 커뮤니티 ↔ 매물/거래 연결

### 현재 문제
- 커뮤니티가 독립적으로 존재
- 매물에 대한 토론이 불가

### 해결
- 매물 상세에 "커뮤니티 토론" 섹션 (해당 매물 관련 글)
- 커뮤니티 글 작성 시 "관련 매물" 태그 가능
- 거래 완료 시 "후기 작성" → 커뮤니티 자동 연결
