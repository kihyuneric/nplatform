# NPLatform — 한국 우선 단기 개발계획서
## Korea-First Priority Plan (2026 Q3 ~ Q4 · 6 개월)

> **목적**: 글로벌 로드맵 (18M / USD 5.6M) 중 **한국 시장에서 즉시 효과가 큰 기능만 추출**
> **원칙**: "있으면 좋다" 제외 · "없으면 거래 안 됨" 만 포함
> **작성일**: 2026-04-29
> **범위**: 6 개월 · 단일 통화 (KRW) · 단일 법역 (한국)

---

## 0. Executive Summary

NPLatform 은 한국 NPL 거래를 위한 **end-to-end 워크플로우**가 이미 구현되어 있습니다.
하지만 **(1) 실 거래 발생** 까지의 마지막 격차 4가지가 남아있습니다 —
**KAMCO 데이터 연동**, **결제 통합 (PG)**, **외부 알림 (SMS·이메일)**, **거래 ML 정확도**.

본 문서는 **6 개월 · 3 FTE · KRW 1.2 억** 규모의 실행 계획으로
**한국 시장에서 첫 100 건 거래** 를 달성하기 위한 우선순위만 정리합니다.

## 1. 한국 시장 현실 점검

### 1.1 한국 NPL 시장의 현재 (2026-04 기준)
- **시장 규모**: KRW 18조 (BIS 2025) — KAMCO 공매가 60% 점유
- **거래 형태**: 80% 가 KAMCO 공매 + 사모AMC 직거래 / 20% 가 매각자 직접 매도
- **디지털 침투율**: 약 30% (대부분 PDF 자료 + 이메일 협상 단계)
- **NPLatform 잠재 시장**: 매도사 직접 매도 + 사모AMC 매도 = **KRW 3.6조**

### 1.2 사용자 페르소나 — 한국 즉시 타겟
| 페르소나 | 규모 | 진입 장벽 | 우선도 |
|---|---|---|---|
| 저축은행·캐피탈·카드사 | 70 사 | KAMCO 외 채널 부족 → **즉시 수요** | **P0** |
| AMC (한국자산관리공사 외 사모) | 30 사 | 데이터 정합 + 분석 자동화 절실 | **P0** |

## 2. 한국에서 "없으면 거래 안 됨" 4가지 격차

### G1. 결제 통합 (PG · ESCROW) — 🔴 최우선
**현재**: ESCROW 안내만 노출 · 실제 결제 처리 미연결
**필요**: 토스페이먼츠 / Inicis / KB ESCROW 3사 통합
**근거**: 회원가입 후 6개월 무료지만, **거래 수수료 결제** 가 막히면 매출 0

### G2. KAMCO · 법원경매 데이터 연동 — 🔴 최우선
**현재**: 매물 53건 (NPLatform 등록만)
**필요**: KAMCO 공매 + 대법원 경매 API/크롤링 → 매물 풀 확장
**근거**: 매물 53건 → 매수자 진입 동기 부족 / KAMCO 1,000+ 건 추가 시 즉시 트래픽 ↑

### G3. SMS · 이메일 외부 알림 — 🟡 즉시 필요
**현재**: 인앱 알림센터만 (사용자 사이트 미접속 시 누락)
**필요**: NCP SENS / Mailgun + 발송 큐 + 옵트인 관리
**근거**: NDA·LOI·정산 임박 등 **시간 민감 알림** 누락 시 거래 실패

### G4. 회수율 ML 정확도 향상 — 🟡 신뢰 구축
**현재**: 통계 추정 (RMSE ±15%)
**필요**: KAMCO 공매 5,000건 backtest + LightGBM v1
**근거**: 사용자가 "AI 분석" 신뢰해야 입찰 → 정확도 ↑ = 사용 빈도 ↑

## 3. 6 개월 로드맵 (3 단계 · 12 Phase)

### **Stage A — 거래 가능 (Month 1~2)**
"매물 등록 → 협상 → 결제 → 정산" 거래 흐름의 마지막 끊긴 고리 연결

| Phase | 명 | 핵심 산출물 | 기간 | 측정 |
|---|---|---|---|---|
| **K-1** | 토스페이먼츠 연동 | 결제 위젯 + 웹훅 + 환불 API | 2 주 | 첫 거래 결제 성공 |
| **K-2** | KB ESCROW 통합 | 보증금 → 잔금 자동 정산 + 인보이스 | 3 주 | 첫 ESCROW 거래 |
| **K-3** | SMS · 이메일 알림 | NCP SENS + Mailgun + 옵트인 | 1 주 | 도달률 95%+ |
| **K-4** | 세금계산서 자동 발행 | 홈택스 API 또는 Bill36524 연동 | 2 주 | 자동 발행률 100% |

### **Stage B — 매물 확보 (Month 3~4)**
"NPLatform 자체 매물 53건" → "한국 공식 NPL 1,000+ 건 풀"

| Phase | 명 | 핵심 산출물 | 기간 | 측정 |
|---|---|---|---|---|
| **K-5** | KAMCO 공매 연동 | 일별 크롤링 + 자동 분석 + 클릭 → KAMCO 이동 링크 | 3 주 | 1,000+ 매물 |
| **K-6** | 대법원 경매 연동 | 일별 크롤링 + 권리분석 자동 (사건번호 기준) | 2 주 | 500+ 사건 |
| **K-7** | 매물 자동 매칭 v2 | 매수 수요 ↔ 외부+내부 매물 풀 합산 | 2 주 | 매칭 정확도 80% |
| **K-8** | 매도사 Bulk Upload | Excel 100건 일괄 등록 + 자동 마스킹 | 1 주 | 평균 25분 → 8분 |

### **Stage C — 신뢰·전환 (Month 5~6)**
"Demo 사용" → "유료 거래 첫 100 건"

| Phase | 명 | 핵심 산출물 | 기간 | 측정 |
|---|---|---|---|---|
| **K-9** | ML 회수율 v1 (KAMCO backtest) | LightGBM + 5K 거래 학습 + 신뢰구간 표기 | 3 주 | RMSE < 12% |
| **K-10** | 모바일 PWA 강화 | 푸시 알림 + 오프라인 캐시 + 홈 화면 추가 | 2 주 | DAU 30% 증가 |
| **K-11** | 1:1 영업 도구 | 매도사 onboarding 추적 + CRM kanban (admin) | 2 주 | 영업 관리 효율 |
| **K-12** | E2E 테스트 + 보안 점검 | Playwright 핵심 시나리오 + 침투 테스트 1회 | 2 주 | 0 critical 이슈 |

## 4. 즉시 실행 (Week 1~4)

## 7. 핵심 메시지 (Pyramid)

> **한국 NPL 시장에서 NPLatform 이 **거래 100 건** 달성하려면**
> **결제 · 외부 매물 · 알림 · ML 정확도 4가지** 만 보강하면 됩니다.
> **6 개월 · 3 FTE · KRW 1.6 억** 으로 실현 가능합니다.

3 가지 즉시 실행 우선순위:
1. **K-1·K-2 결제·ESCROW** (Month 1~2) — 매출 발생의 전제
2. **K-5 KAMCO 연동** (Month 3) — 매물 풀 1,500 건 확보
3. **K-9 ML 정확도** (Month 5~6) — 사용자 신뢰 구축
