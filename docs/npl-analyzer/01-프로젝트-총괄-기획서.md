# 01. NPL 부실채권 투자 분석 플랫폼 — 프로젝트 총괄 기획서

## 1. 프로젝트 비전

### 1.1 한 줄 요약
> "사건번호와 최소 정보만 입력하면, NPL 투자 수익률이 자동 분석되는 대한민국 최초의 부실채권 투자 분석 SaaS"

### 1.2 핵심 가치 제안 (Value Proposition)
| 구분 | 현재 (엑셀) | 우리 플랫폼 |
|------|------------|------------|
| 분석 소요시간 | 4~8시간/건 (수동 엑셀) | 10~15분/건 (입력만 하면 자동) |
| 분석 품질 | 분석가 역량에 의존 | 표준화된 6단계 분석 모델 |
| 시나리오 비교 | 수동으로 셀 복사 | 실시간 다중 시나리오 자동 |
| 민감도 분석 | 별도 시트 수동 작성 | 히트맵 자동 생성 |
| 공유/협업 | 이메일로 엑셀 전송 | URL 공유, 팀 협업 |
| 의사결정 | 숫자만 나열 | 전략 비교 + 리스크 체크리스트 |

### 1.3 타겟 사용자
1. **자산관리회사(AMC)** 분석팀 — 대량 채권 풀 분석
2. **NPL 전문 투자자** — 개별 채권 투자 의사결정
3. **경매 투자자** — 직접 낙찰 수익률 분석
4. **금융기관 여신팀** — 부실채권 매각 시 적정가 산정
5. **법무법인/감정평가사** — 배당 시뮬레이션 검증

### 2.2 아키텍처 원칙
1. **클라이언트 사이드 계산**: 모든 재무 계산은 브라우저에서 실시간 수행. 서버 왕복 없음.
2. **입력/계산 분리**: 사용자 입력(파란셀) → 순수 함수 → 계산 결과(검은셀). 중간에 side effect 없음.
3. **DB는 저장 전용**: Supabase는 입력값과 결과 스냅샷 저장에만 사용. 계산 로직은 DB에 없음.
4. **점진적 입력**: 모든 필드에 기본값 제공. 사용자는 필요한 것만 수정하면 됨.

## 3. 분석 모드 (3가지)

### 3.1 모드 정의

#### 모드 A: 단독물건 경매 분석 (`auction_single`)
- **대상**: 공장, 상가, 토지 등 단일 물건을 직접 낙찰받는 경우
- **분석 내용**: 취득원가 → NOI → Exit 가치 → IRR/NPV/MOIC
- **대응 엑셀**: `경매분석모델_2024타경36029.xlsx`
- **시트 매핑**:
  - ① 기본정보 → Step1 기본정보 폼
  - ② 감정평가 → Step2 감정평가 (선택)
  - ③ 권리분석 → Step3 권리분석
  - ④ 투자분석 → Step4 가정 + Step5 분석결과
  - ⑤ 민감도분석 → Step5 민감도 탭
  - ⑥ 시장통계 → Step5 시장통계 탭 (Phase 3)

#### 모드 B: 단독물건 NPL 분석 (`npl_single`)
- **대상**: 단일 담보물건의 근저당 채권을 매입하는 경우
- **분석 내용**: 배당 워터폴 → 채권 매입 수익률 → 직접 낙찰 IRR → 전략 비교
- **대응 엑셀**: `NPL_밸류에이션모델_2024타경36029.xlsx`
- **시트 매핑**:
  - ① 기본인풋 → Step1 + Step4
  - ② 배당분석 → Step5 배당분석 탭
  - ③ NPL채권매입 → Step5 NPL수익률 탭
  - ④ 직접낙찰IRR → Step5 직접낙찰IRR 탭
  - ⑤ 전략비교 → Step5 전략비교 탭
  - ⑥ 민감도 → Step5 민감도 탭

#### 모드 C: 복수물건 NPL 분석 (`npl_bulk`)
- **대상**: 오피스텔/근생 다수 호실의 병합경매 채권 (KAMCO 매입 등)
- **분석 내용**: 다수 물건 집계 → 배당 워터폴 → 자금조달 구조 → 에쿼티 수익률
- **대응 엑셀**: `npl_통합모델.xlsx`, `npl_재정리.xlsx`
- **시트 매핑**:
  - ①가정 → Step4 가정
  - ②물건현황 → Step2.5 물건현황
  - ③권리·채권 → Step3 권리분석 (채권 분류 포함)
  - ④수익분석 → Step5 분석결과
  - ⑤요약 → Step5 요약 대시보드

#### Sheet ② 배당분석 → 웹 Step5 배당분석 탭
| 엑셀 구조 | 웹 변환 |
|-----------|---------|
| C6~I6: 7개 낙찰가 시나리오 (수동입력) | 시나리오 슬라이더/입력 UI (최대 7개) |
| C10~I10: 낙찰가 복사 | 자동 (입력값 그대로) |
| C11: 경매비용 = 낙찰가 × 비용률 | `dividendWaterfall()` 함수 자동 |
| C12~C14: 당해세, 소액, 임금 차감 | 자동 (가정값 참조) |
| C15: 배당가용금액 = MAX(0, 낙찰-비용-세금...) | 자동 |
| C16~C24: 채권자별 배당 = MIN(채권액, 잔여금) | 자동 (워터폴 순차 계산) |

| C38~J42: 채권 부족액 | 자동 (채권-배당) |

#### Sheet ③ NPL채권매입 → 웹 Step5 NPL수익률 탭
| 엑셀 수식 | 웹 함수 |
|-----------|---------|
| 매입가 = 원금 × (1 - 할인율) | `nplYield.purchasePrice` |
| 순이익 = 배당예상액 - 매입가 | `nplYield.profit` |
| 절대수익률 = 이익/매입가 | `nplYield.absoluteReturn` |
| 연환산 = (1+절대)^(12/월)-1 | `nplYield.annualizedIRR` |
| MOIC = 배당/매입가 | `nplYield.moic` |
| 회수율 = 배당/원금 | `nplYield.recoveryRate` |

#### Sheet ④ 직접낙찰IRR → 웹 Step5 직접낙찰 탭
| 엑셀 수식 | 웹 함수 |
|-----------|---------|
| 총취득원가 = 입찰가+취득세+기타+명도+보수 | `auctionIrr.totalAcquisitionCost` |
| 연간임대 = 단가×면적×12 | `auctionIrr.annualGrossRent` |
| NOI = 임대×(1-공실)×(1-운영비) | `auctionIrr.annualNOI` |
| 현금수익률 = NOI/취득원가 | `auctionIrr.cashYield` |
| Exit A = NOI/ExitCap | `auctionIrr.exitValueCapRate` |
| Exit B = 감정가×(1+상승률)^년 | `auctionIrr.exitValueAppreciation` |
| Exit C = AI시세×상승적용 | `auctionIrr.exitValueAI` |
| CF배열 = [-취득, NOI, ..., NOI+Exit] | `auctionIrr.cashFlows` |
| IRR = IRR(CF배열) | `auctionIrr.irr` (Newton-Raphson) |
| NPV = NPV(10%, CF) + CF[0] | `auctionIrr.npv` |

#### Sheet ④수익분석 → 웹 Step5 (bulk 전용)
| 계산 항목 | 수식 |
|-----------|------|
| 예상 총 낙찰가 | = SUM(물건현황.예상낙찰가) |
| 경매비용 | = 총낙찰가 × 경매비용율 |
| 배당가능액 | = 총낙찰가 - 경매비용 - 선순위합계 |
| 예상 실배당액 | = MIN(배당가능액, NPL최고액합계, NPL채권액배당일) |
| 계약금 | = 매입가 × 계약금비율 |
| 잔금 | = 매입가 × 잔금비율 |
| 이전비용 | = 매입가 × 이전비용율 |
| 비히클사용료 | = 매입가 × 비히클사용료율 |
| 질권대출이자 | = (매입가×LTV) × 이자율 × 회수기간/365 |
| 총 에쿼티 | = 계약금 + 잔금 + 이전비용 + 비히클 + 질권이자 |
| 질권대출금 | = ROUND(매입가 × LTV, -2) |
| 회수금액 | = 실배당액 - 질권대출상환 |
| 수익금 | = 회수금 - 에쿼티 |
| 기간수익률 | = 수익금 / 에쿼티 |
| 연환산수익률 | = 기간수익률 × 365 / 회수기간(일) |
| 손실방어낙찰가 | = 에쿼티 + 선순위 + 경매비용 |
| 손실방어낙찰율 | = 손실방어낙찰가 / 감정가합계 |

## 5. 사용자 흐름 (User Flow)

### 5.1 신규 분석 생성
```
[로그인] → [대시보드] → [+ 새 분석]
  → 분석 모드 선택 (3개 카드 UI)
    ├── "단독물건 경매 분석"
    ├── "단독물건 NPL 분석" (추천)
    └── "복수물건 NPL 분석"
  → [생성] → Step1 기본정보 폼으로 이동
```

### 5.2 Step-by-Step 입력 → 자동 분석
```
Step1 [기본정보] → Step3 [권리분석] → Step4 [가정] → Step5 [분석결과]
  ↑                                                      ↓
  └──── 언제든 이전 Step으로 돌아가서 수정 가능 ←──────────┘
        (수정 시 Step5 결과 즉시 재계산)
```

### 5.3 분석 결과 활용
```
Step5 분석결과
  ├── [배당분석] 탭 → 시나리오별 배당 워터폴 확인
  ├── [NPL수익률] 탭 → 할인율별 수익률 확인
  ├── [직접낙찰] 탭 → IRR/NPV/MOIC 확인
  ├── [전략비교] 탭 → NPL매입 vs 직접낙찰 비교
  ├── [민감도] 탭 → 히트맵으로 최적 투자 포인트 확인
  └── [엑셀 다운로드] → 기존 모델과 동일한 엑셀 생성
```
