# NPLatform UX 사용성 혁신 기획서
## "5초 안에 이해하고, 3번 클릭 안에 도달하고, 실수할 수 없는 UI"

---

## 핵심 원칙

### 3대 사용성 원칙
1. **5초 규칙**: 어떤 페이지든 5초 안에 무엇을 해야 하는지 알 수 있어야 한다
2. **3클릭 규칙**: 원하는 정보까지 최대 3번 클릭으로 도달
3. **실수 방지**: 잘못 누를 수 없고, 잘못 누르더라도 되돌릴 수 있다

---

## 1. 가독성 (Readability) 혁신

### 1.1 타이포그래피 시스템
```
현재 문제:
  - 금액이 "850000000"처럼 표시되면 읽기 어려움
  - 한국어와 숫자 혼용 시 시각적 우선순위 불명확
  - 작은 화면에서 텍스트 잘림

해결:

금액 표시 통일:
  ❌ 850,000,000원
  ❌ 850000000
  ✅ 8억 5,000만원         ← 주요 금액 (큰 텍스트)
  ✅ 8.5억                 ← 요약/카드 (압축 표시)
  ✅ 850,000,000원 (상세)   ← 툴팁 hover 시 원본

비율 표시:
  ✅ LTV 70.8%             ← 숫자 + 프로그레스바
  ✅ 할인율 29.2% ▼         ← 방향 화살표 포함
  ✅ 낙찰가율 87.3% ▲       ← 색상 + 방향

날짜 표시:
  ✅ 오늘 14:32             ← 당일
  ✅ 어제                   ← 전일
  ✅ 3일 전                 ← 7일 이내
  ✅ 2026.03.15             ← 7일 이후

폰트 계층:
  페이지 타이틀: 28px / bold / #1B3A5C
  섹션 타이틀: 20px / semibold / #1B3A5C
  카드 타이틀: 16px / semibold / gray-900
  본문: 14px / normal / gray-700
  보조 텍스트: 12px / normal / gray-500
  라벨: 11px / medium / gray-400 / uppercase
```

### 1.2 정보 밀도 최적화
```
원칙: "한 화면에 최대 7±2개 정보 그룹"

매물 카드 정보 계층:
  Layer 1 (즉시 인식): 금액, 상태 배지, AI등급
  Layer 2 (1초 후): 주소, 담보유형, 기관명
  Layer 3 (관심 시): LTV, 할인율, 조회수
  Layer 4 (hover/클릭): 상세 정보, 분석 링크

테이블 정보 계층:
  고정 컬럼 (항상 보임): 번호, 주소, 상태
  스크롤 컬럼 (필요 시): 나머지 데이터
  요약 행 (하단 고정): 평균/합계
```

### 1.3 색상 의미 체계
```
상태 색상 (일관된 의미):
  🟢 #10B981 (emerald) = 긍정 (활성, 승인, 수익, AI A등급)
  🔵 #3B82F6 (blue)    = 진행중 (입찰중, 심사중, 분석중)
  🟡 #F59E0B (amber)   = 주의 (마감임박, 리스크 중, AI C등급)
  🔴 #EF4444 (red)     = 위험/종료 (마감, 거부, 손실, AI D등급)
  ⚪ #6B7280 (gray)    = 비활성 (초안, 대기, 미확인)

금액 색상:
  수익: #10B981 (초록)
  손실: #EF4444 (빨강)
  중립: #1B3A5C (네이비)

배경 색상:
  페이지: #FAFBFC (라이트 그레이)
  카드: #FFFFFF
  강조 섹션: 그라데이션 (#1B3A5C → #2E75B6)
  경고 섹션: #FFFBEB (연한 노랑)
```

## 2. 가시성 (Visibility) 혁신

### 2.1 시각적 계층 (Visual Hierarchy)
```
모든 페이지 공통 구조:

┌─────────────────────────────────────┐
│ 🔷 히어로 바 (현재 위치, 핵심 액션)    │ 48px, gradient
├─────────────────────────────────────┤
│ 📊 KPI 카드 (핵심 숫자 3~6개)         │ 카드 그리드
├─────────────────────────────────────┤
│ 🔍 필터/검색 바 (sticky)              │ 상단 고정
├─────────────────────────────────────┤
│ 📋 메인 콘텐츠                        │ 테이블/카드
├─────────────────────────────────────┤
│ 📄 페이지네이션 / 더보기               │
└─────────────────────────────────────┘

히어로 → KPI → 필터 → 콘텐츠 순서로 시선 유도
각 섹션 간 24px 여백으로 구분
```

### 2.2 CTA (Call-to-Action) 가시성
```
원칙: "가장 중요한 버튼이 가장 눈에 띄어야 한다"

Primary CTA:
  - 배경: #1B3A5C (solid)
  - 크기: h-12 px-6 text-base
  - 위치: 우상단 또는 섹션 우측
  - 예: "매물 등록", "입찰 참여", "분석 시작"

Secondary CTA:
  - 배경: #10B981 (accent)
  - 크기: h-10 px-4 text-sm
  - 예: "PDF 다운로드", "엑셀 내보내기"

Tertiary CTA:
  - 배경: transparent, border
  - 크기: h-9 px-3 text-sm
  - 예: "더보기", "필터 초기화"

Floating CTA (모바일):
  - 화면 하단 고정 (safe area 위)
  - 핵심 액션 1개만 (예: "입찰 참여")
  - 그림자 + 둥근 모서리
  - 스크롤 시 축소/확대 애니메이션
```

### 2.3 상태 표시 강화
```
Empty State (데이터 없을 때):
  - 큰 아이콘 (64px, 연한 색)
  - 명확한 메시지 ("검색 결과가 없습니다")
  - 구체적 안내 ("필터를 변경하거나 검색어를 수정해보세요")
  - 액션 버튼 ("필터 초기화", "전체 매물 보기")

Loading State:
  - 스켈레톤 (실제 콘텐츠와 동일한 레이아웃)
  - 펄스 애니메이션 (shimmer effect)
  - 진행률 표시 (가능한 경우)

Error State:
  - 빨간 아이콘 + 명확한 에러 메시지
  - "다시 시도" 버튼
  - 기술적 메시지 숨김 (접기/펼치기)

Success State:
  - 체크 아이콘 + 초록 배경
  - 자동 사라짐 (3초 후)
  - 다음 단계 안내
```

### 2.4 데이터 시각화 가시성
```
숫자 → 시각적 표현 변환:

LTV 70.8%:
  ┌──────────████████░░┐ 70.8%
  파랑(~60%) → 주황(60~80%) → 빨강(80%~)

AI등급 A:
  ◉ A  (큰 원, 초록 배경, 흰 글씨)

리스크 점수 35/100:
  ████████████░░░░░░░░ 35 (초록 = 낮은 리스크)

할인율 29.2%:
  ▼ 29.2%  (아래 화살표, 초록 = 투자자에게 유리)

조회수 245:
  👁 245  (아이콘 + 숫자, 회색)

D-Day:
  D-3  (빨간 배경, 깜빡임)
  D-15 (주황 배경)
  D-30 (회색 배경)
```

## 3. 사용 편의성 (Usability) 혁신

### 3.1 온보딩 가이드
```
첫 방문자 경험:

Step 1 - 웰컴 모달:
  "NPLatform에 오신 것을 환영합니다!"
  [매물 찾기 (투자자)] [매물 등록 (기관)] [둘러보기]
  → 선택에 따라 다른 가이드 시작

Step 2 - 투어 가이드 (선택 시):
  반투명 오버레이 + 하이라이트 박스
  "이곳에서 NPL 매물을 검색할 수 있습니다" → 다음
  "AI가 자동으로 분석한 결과를 확인하세요" → 다음
  "관심 매물을 저장하고 비교해보세요" → 완료

Step 3 - 체크리스트 위젯:
  사이드바 또는 플로팅 위젯
  □ 프로필 완성하기
  □ 첫 매물 검색하기
  □ AI 분석 해보기
  □ 관심매물 저장하기
  ■ 4/4 완료! 🎉

기술:
  localStorage 'onboarding_completed' 플래그
  components/onboarding/tour-guide.tsx
  components/onboarding/checklist-widget.tsx
```

### 3.2 컨텍스트 도움말 시스템
```
용어 설명 (인라인):
  "LTV ⓘ" ← hover 시 "Loan-to-Value. 대출금액 ÷ 감정가 × 100"
  "낙찰가율 ⓘ" ← "경매에서 실제 낙찰된 가격 ÷ 감정가 × 100"
  "NDA ⓘ" ← "비밀유지계약. 서명 후 상세 정보 열람 가능"

구현:
  <Tooltip> 래핑된 <InfoIcon> 컴포넌트
  components/ui/help-text.tsx:
    <HelpText term="LTV">70.8%</HelpText>
    → 숫자 옆에 작은 ⓘ 아이콘, hover 시 설명

용어 사전 연결:
  각 설명 하단 "자세히 보기 →" 링크 → /knowledge/glossary#ltv
```

### 3.3 스마트 폼 UX
```
원칙: "사용자가 입력하기 전에 미리 알려주고, 입력 중에 피드백하고, 입력 후에 확인한다"

금액 입력:
  입력: 850000000
  실시간 변환 표시: "= 8억 5,000만원"
  유효성: 초록 체크 (정상) / 빨간 X (범위 초과)

주소 입력:
  시도 선택 → 시군구 자동 필터 → 읍면동 자동 필터
  입력 시 자동완성 드롭다운

날짜 입력:
  캘린더 피커 + 빠른 선택 (오늘, 내일, 1주후, 1개월후)
  과거 날짜 비활성화 (입찰 마감일)

전화번호:
  자동 하이픈: 010-1234-5678

사업자번호:
  자동 하이픈: 123-45-67890
  실시간 형식 검증

모든 필수 필드:
  라벨 옆 빨간 * 표시
  미입력 시 부드러운 흔들림 애니메이션
  에러 메시지: 필드 바로 아래 (빨간 텍스트)
```

### 3.4 되돌리기/실행취소
```
기능:
  - 매물 삭제 → "삭제되었습니다" 토스트 + "되돌리기" 버튼 (5초간)
  - 관심매물 해제 → "관심 해제" 토스트 + "되돌리기" (3초간)
  - 필터 변경 → "이전 필터로 돌아가기" 링크
  - 폼 작성 중 이탈 → "저장하지 않은 변경사항이 있습니다" 확인 모달

구현:
  - Sonner 토스트 + action 버튼
  - beforeunload 이벤트
  - 폼 dirty state 감지
```

### 3.5 키보드 단축키
```
글로벌:
  Cmd/Ctrl + K → 글로벌 검색 팔레트
  Cmd/Ctrl + / → 단축키 도움말
  Escape → 모달/드롭다운 닫기

검색 페이지:
  / → 검색 입력란 포커스
  F → 필터 패널 토글
  E → 엑셀 다운로드
  T → 테이블/카드 뷰 전환
  1~9 → 탭 전환 (개요~경공매)

매물 상세:
  L → 관심매물 토글
  C → 비교함에 담기
  P → PDF 다운로드
  B → 입찰 참여

시뮬레이터:
  Enter → 계산 실행
  S → 시나리오 저장
  R → 리셋

표시:
  페이지 하단 "⌨ 단축키" 버튼 → 단축키 목록 Sheet
```

### 3.6 스마트 검색
```
검색 입력 UX 개선:

자연어 검색:
  "강남 아파트 10억 이하" → 자동 파싱
    지역: 강남 / 담보유형: 아파트 / 감정가: ~10억
  "서울 상가 AI A등급" → 자동 파싱
    지역: 서울 / 담보유형: 상가 / AI등급: A

자동완성:
  입력 중 → 주소, 기관명, 담보유형 제안 드롭다운
  최근 검색어 → 회색 히스토리 항목
  인기 검색어 → "🔥 인기" 배지

검색 결과 하이라이트:
  검색어와 매칭되는 부분 노란 하이라이트
  "강남" 검색 → 주소에서 "강남" 강조 표시
```

### 3.7 진행률/상태 시각화
```
거래 프로세스:
  ①관심 ─②NDA─ ③실사 ─④협상─ ⑤LOI─ ⑥계약─ ⑦완료
  ●────●────●────○────○────○────○
  완료       진행중    미시작

입찰 등록 위자드:
  Step 1 ─ Step 2 ─ Step 3 ─ Step 4
  ✓────── ✓────── ●────── ○
  완료     완료    현재    미시작

  하단: "4단계 중 3단계 (75% 완료)"

파일 업로드:
  ████████████░░░░░░░░ 60%
  "3/5 파일 처리 완료"

분석 진행:
  기본정보 수집... ✓
  가치평가 분석중... ⟳
  유사사례 검색... ○
  리포트 생성... ○
```

### 3.8 알림/피드백 시스템
```
알림 계층:

Critical (즉시 주목):
  - 풀스크린 모달 + 빨간 배경
  - 예: "입찰이 마감되었습니다"
  - 닫기 전 반드시 확인

Important (주목 유도):
  - 토스트 상단 + 주황 배경
  - 예: "새 입찰이 접수되었습니다"
  - 5초 후 자동 닫힘 + 액션 버튼

Info (참고):
  - 토스트 하단 + 파란 배경
  - 예: "검색 결과 30건"
  - 3초 후 자동 닫힘

Success (확인):
  - 토스트 하단 + 초록 배경 + 체크 아이콘
  - 예: "저장되었습니다"
  - 2초 후 자동 닫힘

촉각 피드백 (모바일):
  - 버튼 클릭: 짧은 진동 (10ms)
  - 입찰 성공: 강한 진동 (50ms)
  - 에러: 패턴 진동 (짧-짧-긴)
```

## 4. 페이지별 사용성 혁신 상세

### 4.1 검색 페이지 UX 혁신
```
Before (현재):
  페이지 로드 → 40건 테이블 → 필터 열기 → 조건 설정 → 적용

After (혁신):
  페이지 로드 → 자연어 검색바 중심
  "무엇을 찾으시나요?" placeholder
  인기 태그: #강남아파트 #10억이하 #AI_A등급 #입찰중
  입력 → 실시간 결과 카운트 "32건 매칭"
  Enter → 결과 표시 (기본: 카드 뷰)

  필터: 사이드 패널 (데스크탑) / 바텀시트 (모바일)
  적용된 필터: 상단 칩 (개별 제거 가능)
  정렬: 드롭다운 대신 탭형 ("관련도순 | 최신순 | 금액순")
```

### 4.2 매물 상세 UX 혁신
```
Before:
  7개 탭 → 탭 클릭 → 정보 확인 → 다른 탭 클릭

After:
  원페이지 스크롤 (탭 대신 섹션)
  좌측: 섹션 네비게이션 (sticky, 현재 위치 하이라이트)
  스크롤 시 자동 섹션 감지 (IntersectionObserver)

  우측 사이드바 (sticky):
    큰 "입찰 참여" 버튼
    가격 요약 카드
    관심/비교/공유 아이콘 행

  모바일:
    하단 고정 바: [관심❤️] [입찰참여] [공유↗]
    섹션 점프: 상단 드롭다운
```

### 4.3 시뮬레이터 UX 혁신
```
Before:
  좌(입력폼)+우(결과) → 스크롤 필요 → 차트 멀리

After:
  Step-by-step 모드 (기본):
    1. 물건 선택 (프리셋 또는 직접 입력)
    2. 비용 설정 (슬라이더 + 실시간 미리보기)
    3. 결과 확인 (차트 + 테이블 + AI 의견)
    "이전 | 다음" 네비게이션

  Expert 모드 (토글):
    현재와 동일한 분할 레이아웃
    모든 옵션 한 화면에

  핵심 개선:
    입력값 변경 → 결과 즉시 업데이트 (debounce 300ms)
    슬라이더 드래그 → 차트 실시간 반응
    최적 입찰가 하이라이트 (녹색 깜빡임)
```

### 4.4 입찰 참여 UX 혁신
```
Before:
  입찰 모달 → 금액 입력 → NDA 체크 → 제출

After:
  3단계 플로우:

  Step 1 - 매물 확인:
    핵심 정보 요약 카드 (감정가, 최저가, 현재 최고입찰가)
    "정말 이 매물에 입찰하시겠습니까?"
    [취소] [다음 →]

  Step 2 - 금액 입력:
    큰 숫자 입력 (tel 키패드)
    퀵 버튼: 감정가의 [60%] [70%] [80%] [90%]
    실시간: "= 5억 9,500만원"
    실시간: "감정가 대비 70.0%"
    실시간: "현재 최고가 대비 +5,000만원"
    [이전] [다음 →]

  Step 3 - 최종 확인:
    요약: 매물, 금액, 비율
    NDA 동의 체크 (이미 서명했으면 "서명 완료 ✓")
    [이전] [입찰 제출]

  제출 후:
    성공 화면 (초록 체크 + 축하 메시지)
    "입찰 현황 보기" | "다른 매물 보기"
```
