### 🤖 I1. AI 문서 자동화 엔진 — 25점

> OCR로 서류를 읽고, AI가 매물 정보를 자동 입력하고, 계약서를 자동 생성

#### I1-1. OCR 자동 입력 (10점)
```
기능:
  - 등기부등본 PDF 업로드 → AI가 자동 파싱
    → 소유자, 주소, 면적, 근저당, 설정금액, 채권자 자동 추출
  - 감정평가서 PDF → 감정가, 토지/건물가, 용도지역 추출
  - 법원 매각공고 → 사건번호, 법원, 매각기일, 최저가 추출
  - 추출 결과 → 매물 등록 폼 자동 채움 (사용자 확인 후 저장)

페이지:
  /seller/listings/new → Step 0 "서류 업로드" 추가
  사용자가 서류 올리면 → AI 파싱 → 나머지 5단계 자동 채움

기술:
  - pdf-parse (텍스트 추출)
  - Claude API (구조화 파싱 프롬프트)
  - 신뢰도 표시 (높음/중간/낮음, 사용자 수정 가능)

API:
  POST /api/v1/ocr/parse — PDF 업로드 → 구조화 JSON 반환
```

#### I1-2. AI 계약서 자동 생성 (8점)
```
기능:
  - 거래 조건 입력 → AI가 매매계약서 초안 생성
  - 템플릿: NPL 양수도계약서, 채권매매계약서, NDA
  - 변수 자동 치환 (당사자, 물건, 금액, 조건, 일자)
  - DOCX/PDF 다운로드
  - 버전 관리 (v1, v2, 최종본)

페이지:
  /deal-rooms/[id]/contract — 계약서 생성 탭

기술:
  - docx 라이브러리 (이미 설치됨)
  - 템플릿 기반 + AI 커스터마이징
```

#### I1-3. AI 실사 리포트 자동 생성 (7점)
```
기능:
  - 매물 데이터 + 외부 데이터 종합 → 실사 리포트 자동 생성
  - 항목: 법률 검토, 담보 평가, 재무 분석, 점유 현황, 리스크 요인
  - 각 항목 AI 자동 평가 (양호/주의/위험) + 근거 텍스트
  - PDF 리포트 다운로드 (10~15페이지)

페이지:
  /buyer/due-diligence/[id] — AI 실사 리포트 뷰어

API:
  POST /api/v1/due-diligence/generate — AI 리포트 생성
```

### ⚡ I2. 실시간 라이브 경매 시스템 — 25점

> 온라인 경매장에서 실시간으로 입찰하고 경쟁하는 라이브 경매

#### I2-1. 라이브 경매 방 (12점)
```
기능:
  - 매도자가 "라이브 경매" 일정 등록 (시작 시간, 진행 시간)
  - 경매 시작 → 참여자 실시간 입장
  - 실시간 입찰 (Supabase Realtime WebSocket)
  - 현재 최고가 실시간 업데이트
  - 입찰 히스토리 실시간 표시 (익명: 참여자1, 2, 3...)
  - 카운트다운 타이머 (마감 30초 전 자동 연장)
  - 낙찰 확정 → 결과 화면 → 딜룸 자동 생성
  - 참여자 수 실시간 표시

페이지:
  /marketplace/live-auction — 예정된 라이브 경매 목록
  /marketplace/live-auction/[id] — 라이브 경매 방

UI:
  - 좌: 매물 정보 카드 (사진, 감정가, 최저가)
  - 중: 입찰 현황 (현재가, 입찰 히스토리 스크롤)
  - 우: 내 입찰 패널 (금액 입력, 원클릭 +100만/+500만/+1000만)
  - 하단: 참여자 수, 남은 시간, 채팅

기술:
  - Supabase Realtime (broadcast channel)
  - Optimistic UI (입찰 즉시 반영, 서버 확인 후 정정)
```

#### I2-2. 경매 관리 (8점)
```
매도자 측:
  - 라이브 경매 등록 (매물 선택, 시작가, 일시, 진행시간)
  - 경매 중 모니터링 (관리자 뷰)
  - 경매 결과 리포트 (참여자수, 입찰횟수, 최종가, 경쟁도)

관리자 측:
  - 경매 승인/반려
  - 라이브 모니터링 (여러 경매 동시)
  - 부정 입찰 감지 (동일 IP, 비정상 패턴)
```

#### I2-3. 경매 캘린더 + 알림 (5점)
```
페이지:
  /marketplace/calendar — 월간/주간 캘린더 뷰
    라이브 경매 일정 + 일반 입찰 마감일 통합
    필터: 담보유형, 지역, 금액대
    구글 캘린더 연동 (ics 다운로드)

알림:
  경매 시작 30분 전 알림
  관심 매물 경매 등록 시 알림
```

### 📱 I3. 모바일 PWA + 오프라인 — 20점

> 설치 없이 네이티브 앱처럼 사용, 오프라인에서도 핵심 기능 동작

#### I3-1. PWA 설정 (8점)
```
기능:
  - manifest.json (앱 이름, 아이콘, 테마색, 시작 URL)
  - Service Worker (next-pwa)
  - 홈화면 추가 프롬프트 (iOS/Android)
  - 앱 아이콘 (192x192, 512x512)
  - 스플래시 스크린
  - 상태바 테마 (#1B3A5C)

파일:
  public/manifest.json
  public/sw.js (자동 생성)
  public/icons/ (앱 아이콘 세트)
```

#### I3-2. 오프라인 기능 (7점)
```
기능:
  - 관심매물 오프라인 열람 (IndexedDB 캐시)
  - 최근 분석 결과 오프라인 열람
  - 시뮬레이터 완전 오프라인 동작 (계산은 클라이언트)
  - 네트워크 상태 표시 (온라인/오프라인 배너)
  - 오프라인 중 작업 → 온라인 복귀 시 자동 동기화

기술:
  - IndexedDB (idb 라이브러리)
  - Background Sync API
  - Cache-first 전략 (정적 자산)
  - Network-first 전략 (API 데이터)
```

#### I3-3. 모바일 최적화 (5점)
```
기능:
  - 터치 제스처 (스와이프 뒤로가기, 당겨서 새로고침)
  - 바이브레이션 피드백 (입찰 성공, 알림)
  - 푸시 알림 (Web Push API)
  - 카메라 접근 (서류 촬영 → OCR 즉시 파싱)
  - 생체인증 (Face ID/지문 mock UI)
```

### 🔗 I5. 외부 데이터 연동 API — 15점

> 공공데이터, 법원경매, 등기부 연동으로 데이터 자동화

#### I5-1. 공공데이터 연동 (8점)
```
기능:
  - 국토교통부 실거래가 API 연동 (mock)
    → 매물 주변 실거래가 자동 표시
  - 건축물대장 API (mock)
    → 면적, 용도, 건축년도 자동 채움
  - 공시가격 API (mock)
    → 공시지가, 공시가격 자동 표시
  - 법원 경매 정보 API (mock)
    → 사건번호로 경매 진행상황 자동 조회

UI:
  매물 상세 "외부 데이터" 탭
  자동 채움 시 출처 표시 (국토부, 법원)

API:
  GET /api/v1/external/real-price?address=...
  GET /api/v1/external/building-register?address=...
  GET /api/v1/external/official-price?address=...
  GET /api/v1/external/court-auction?case_number=...
```

#### I5-2. 데이터 내보내기 API (7점)
```
기능:

  - Rate limiting (100 req/min)
  - API 문서 자동 생성 (Swagger UI mock)

페이지:
  /developer/api-docs — API 문서
  /developer/api-keys — API 키 관리

기술:
  - API 키 인증 미들웨어
  - 요청 로깅 + 사용량 대시보드
```
