**종합 점수: 62/100**

### 핵심 문제점
1. **메뉴 체계 혼란** — 7개 드롭다운이 너무 많고, 사용자 역할별 최적화 안 됨
2. **가이드 접근성 0** — 메뉴에 있지만 눈에 안 띔, 첫 사용자 유도 없음
3. **다국어 없음** — 글로벌 서비스 불가
4. **모바일 최적화 부족** — 탭바만 있고, 앱 수준 UX 아님
5. **데이터 mock** — 실제 DB 연동이 안 된 페이지 많음

## 4부: 다국어 (i18n) 전략

### 구현 방식: next-intl

```
app/
  [locale]/
    (main)/
      page.tsx
      exchange/
      ...
messages/
  ko.json  -- 한국어 (기본)
  en.json  -- 영어
  ja.json  -- 일본어
```

### 언어 전환 UI
- 상단 네비게이션 우측: 🌐 KO ▼ → KO/EN/JP 드롭다운
- 언어 변경 시 URL 변경: `/ko/exchange` ↔ `/en/exchange` ↔ `/ja/exchange`
- 쿠키에 선호 언어 저장

### 번역 범위
| 단계 | 범위 | 예상 키 수 |
|------|------|-----------|
| Phase 1 | 네비게이션, 공통 UI, 가이드 | ~500 |
| Phase 2 | 전체 페이지 텍스트 | ~3,000 |
| Phase 3 | AI 생성 콘텐츠 자동 번역 | 동적 |

## 5부: 모바일 최적화 + 앱 전략

### 모바일 반응형 (현재 → 개선)

| 항목 | 현재 | 개선 |
|------|------|------|
| 탭바 | 5개 (홈/마켓/허브/분석/더보기) | 역할별 맞춤 (매수자/매도자 다른 탭) |
| 드로어 | 기본 | 풀스크린 네비게이션 + 검색 |
| 스와이프 | 없음 | 매물 카드 좌우 스와이프 (관심/패스) |
| 제스처 | 없음 | 풀다운 새로고침, 바텀시트 |
| 푸시 알림 | 미구현 | Web Push + FCM 연동 |

### 앱 전환 로드맵

```
현재: Next.js PWA (웹앱)
  ↓ Phase 1: PWA 완성 (오프라인, 푸시, 홈 추가)
  ↓ Phase 2: React Native 또는 Capacitor로 네이티브 래핑
  ↓ Phase 3: 네이티브 기능 추가 (카메라 OCR, 생체인증, 지도 AR)
```

## 6부: 외부 데이터 연동 전략

### 사용자의 기존 부동산 데이터 서비스 연동

| 데이터 | 연동 방식 | 활용 |
|--------|----------|------|
| 실거래가 시세 | REST API 또는 DB 직접 연결 | NPL 적정가 산출, 담보물 시세 비교 |
| 부동산 통계 | API | 시장 인텔리전스 대시보드 |
| 뉴스 기사 | RSS/API | 뉴스 피드 자동 수집 |
| 경매 데이터 | 크롤링/API | 경매 시뮬레이터, 입찰 정보 |
| 등기부 정보 | API | 실사 자동 검증 |
| 지역 데이터 | GeoJSON | 지도 히트맵, 지역 분석 |

### 연동 아키텍처

```
[기존 부동산 서비스] ──API──→ [NPLatform 중간 서버]
                                    ↓
                              [Supabase DB]
                                    ↓
                              [NPLatform 프론트엔드]
```

### 구현 파일
- `lib/external-data.ts` — 외부 데이터 연동 클라이언트
- `api/v1/external/real-estate/route.ts` — 실거래가 프록시
- `api/v1/external/news/route.ts` — 뉴스 피드 프록시
- `api/v1/external/auction/route.ts` — 경매 데이터 프록시

## 7부: 실행 로드맵

### Phase 1: 즉시 (1~2주)
- [ ] 메뉴 체계 재구성 (6개 카테고리)
- [ ] 가이드 접근성 강화 (메인 배너 개선, 누락 페이지 GuideButton 추가)
- [ ] 다국어 기반 설정 (next-intl 설치, KO 기본, 언어 선택 UI)
- [ ] 모바일 탭바 역할별 맞춤
- [ ] 모든 역할 체크에 관리자 슈퍼계정 허용 (완료됨)

### Phase 2: 핵심 연동 (2~4주)
- [ ] Supabase 마이그레이션 실제 실행 (37 테이블)
- [ ] Mock → 실제 DB 전환 (핵심 API 30개)
- [ ] 기존 부동산 데이터 서비스 API 연동
- [ ] 뉴스 피드 자동 수집
- [ ] PG사 연동 (토스페이먼츠)

### Phase 3: 글로벌 준비 (4~6주)
- [ ] 영어 번역 완료
- [ ] 일본어 번역 완료
- [ ] PWA 완성 (오프라인, 푸시 알림)
- [ ] 모바일 UX 고도화 (스와이프, 바텀시트)
- [ ] SEO 국제화 (hreflang, 언어별 sitemap)

### Phase 4: 앱 + 확장 (6~12주)
- [ ] React Native 또는 Capacitor 네이티브 앱
- [ ] 카메라 OCR (모바일 문서 스캔)
- [ ] 생체인증 (Face ID, 지문)
- [ ] AR 지도 (담보물 현장 확인)
- [ ] AI 에이전트 고도화 (자연어 대화형 거래)

### Phase 5: 글로벌 론칭
- [ ] 일본 NPL 시장 진출 (일본어 + 현지 규제 대응)
- [ ] 동남아 확장 (영어 기반)
- [ ] 유럽 시장 조사
- [ ] 글로벌 파트너십 (DebtX, Debexpert 등)
